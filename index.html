Bulk Uploader

							
Instructions:

							
    
								
  1. Prepare a csv document with a column for each variable kind you would like stored (i.e, Artist, Title, Description, filename). Please include an artifact id column or the row numbers will be used as the id
    2. 
								
  2. Inside your csv document, for the "files" column (where you set your file names and such) should use the following format if individual pricing info is desired. To see how to format each file in the array, take a look here at artifact.storage.files it contains all the currently allowed variables for the file information including pricing. If you do not care about per file payment info, please just include an array containing strings with the filenames, you also may define columns for the price. Please note that this price will be applied to all files, i.e. if you have a 1¢ play fee, it would apply to all files without requiring you to set that for each individual file.
    3. 
								
  3. Place all files into the same folder and name all of your files according to the following format: {artifact-id}.{filename}.{extension} aka 9c7Ajc.album_artwork.jpg
    4. 
								
  4. Click or Drag and drop the folder containing all of your files to bulk upload onto the dropzone below.
    5. 
								
  5. Select your .csv file
    6. 
								
  6. Assign JSON locations for each column. After you select your .csv file, you should see input boxes show up below the uploader. Each of these input values are linked to one of your Columns in your csv. Input where in the artifact you would like that data variable to be stored. For example, if I had a column named "Artist Name" I would want to link that to artifact.info.extraInfo.artist as that is how you would reference that variable in the JSON object. Curious what variables are commonly supported? You can check out the official OIP Media Protocol standard here.
